interface MultipartEntity

Represents an entity that contains multiple EntityParts.

The natural uses for multipart entities is for entities to have specific hitboxes with damage multipliers or for large entities to have more accurate hitboxes.

NOTE: When instantiating EntityParts, on the client, make sure to call setId. 



public void onSpawnPacket(EntitySpawnS2CPacket packet) {
    super.onSpawnPacket(packet);
    var entityParts = this.getEntityParts();

    // We make sure not to override the base entity id
    // Mojang did this on the ender dragon, and it caused very janky hitboxes
    for(int i = 1; i <= entityParts.length; i++) {
        entityParts[i].setId(i + packet.getId());
    }
}
