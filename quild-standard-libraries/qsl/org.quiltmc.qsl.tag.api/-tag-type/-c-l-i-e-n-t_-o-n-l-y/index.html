CLIENT_ONLY

Represents a client-only tag type, tags are loaded from resource packs' assets directory instead of the data directory.

Those tags are not present on the server, thus no syncing will happen.

A net.minecraft.registry.tag.TagKey with this type will be considered entirely different from a net.minecraft.registry.tag.TagKey with any of the other types even if they both share the same identifier, which means both keys won't link to the same tag content, client tag content will not leak into other tags, and vice-versa.
