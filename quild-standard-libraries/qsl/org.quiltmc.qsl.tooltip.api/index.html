The Quilt Tooltip API

What are the tooltip APIs?

With Minecraft 1.17, a new system has been introduced through the net.minecraft.item.BundleItem: custom tooltip components.

Those tooltip components are created using mainly two classes:

  • net.minecraft.client.item.TooltipData, which holds information about the tooltip, exists on both sides;
  • net.minecraft.client.gui.tooltip.TooltipComponent, which does the rendering of the tooltip, only exists on the client.
An item can return a custom tooltip data by overriding the method getTooltipData. However, there is no method in Vanilla to convert custom tooltip data into a component.

Thus this API introduces:

  • org.quiltmc.qsl.tooltip.api.ConvertibleTooltipData to provide the missing conversion mechanism directly in the tooltip data, please read its documentation carefully;
  • org.quiltmc.qsl.tooltip.api.client.TooltipComponentCallback on the client, an event which is triggered when a tooltip data tries to be converted to a tooltip component.

Text-based tooltips

When hovering an item, the client can display text about it in a tooltip. It can be appended to per-item through appendTooltip, but what if you want to append new lines on an existing item?

That's the job of org.quiltmc.qsl.tooltip.api.client.ItemTooltipCallback.
