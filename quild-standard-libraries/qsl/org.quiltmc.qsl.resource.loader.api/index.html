The Resource Loader and its APIs.

Quick note about vocabulary in Resource Loader and Minecraft:

  • Resource Pack refers to both client-sided resource pack and data pack.
  • Virtual Resource Pack refers to a resource pack that may be generated at runtime, or simply doesn't exist directly on disk.
  • Group Resource Pack refers to a virtual resource pack that groups multiple resource packs together.

Modded Resource Pack Handling

The Resource Loader will create a resource pack for each mod that provides resources in assets or data top-level directories of the mod. Those mod resource packs are grouped into the default resource pack. They can override the default resource pack, but it is not recommended as it forcefully introduce changes that users may not want, instead using a built-in resource pack would be better. Built-in Mod Resource Pack The Resource Loader adds manually registered mod resource packs. Those resource packs are registered with registerBuiltinResourcePack, or registerBuiltinResourcePackResource Pack InjectionResource Pack Profile Provider The Resource Loader gives a method to register ResourcePackProviders, which may be used to add new resource packs that are visible to the player in the resource pack selection screens or the datapack command. Virtual Resource Packs Some mods may need to rely on virtual resource packs to generate resources on the fly. The Resource Loader provides utilities to work with such kind of resource packs:
  • getRegisterDefaultResourcePackEvent - an event to register resource packs that are injected into the default resource pack.
  • getRegisterTopResourcePackEvent - an event to register resource packs that are on the top of the resource pack hierarchy, those resource packs are invisible to the player.
  • org.quiltmc.qsl.resource.loader.api.InMemoryResourcePack - a resource pack implementation whose resources are stored in the live memory.
  • org.quiltmc.qsl.resource.loader.api.GroupResourcePack - a resource pack implementation which can be used to group multiple resource packs into one.
Extending Vanilla Built-In Resource Packs The Resource Loader will inject resources into vanilla built-in resource packs for each mod that provides the resources in a top-level directory inside the mod with the pack's raw name (for example, programmer_art), whose structure is similar to a normal resource pack.

The currently supported targets are:

  • Programmer Art (programmer_art)
  • High Contrast ({@value net.minecraft.client.resource.ClientBuiltinResourcePackProvider#HIGH_CONTRAST_NAME})

Do note that this won't allow for overriding resources that the built-in resource packs already have.

In the case of data-packs, Vanilla has a data/minecraft/datapacks directory in which folders for each built-in data-pack is present, using the same paths in mods will allow to extend those data-packs as well.

Resource Reloaders

The Resource Loader allows mods to register resource reloaders through registerReloader, which are triggered when resources are reloaded. Resource reloaders can define a specific ordering through addReloaderOrdering, vanilla resource reloader identifiers may be found in org.quiltmc.qsl.resource.loader.api.reloader.ResourceReloaderKeys.
